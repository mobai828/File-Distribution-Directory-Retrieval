# 自动分件与全文检索系统 (Auto-Filing & Full-Text Search System)

## 项目简介

本项目是一个基于 FastAPI 开发的轻量级数字档案管理系统原型。它模拟了企业级档案系统中最核心的两个业务流程：
1. **自动分件与路径回写 (Auto-Filing & Writeback)**：将无序上传的离散扫描件图片，根据档号规则自动存入对应的多级物理文件夹，并记录到数据库中，实现“账物相符”。
2. **全文检索 (Full-Text Search)**：借助 Elasticsearch 的强大倒排索引，对档案元数据及 OCR 长文本进行毫秒级模糊搜索、高亮展示及规则扩展。

---

## 核心功能特性

### 📁 1. 自动分件与物理落盘
- **智能解析**：通过强大的正则表达式，支持解析类似 `KJ-JJ-2017-02-001` 或 `WS·2024·D10-0311-001.jpg` 这样包含横杠或中间点分隔符的复杂档号格式。
- **动态建树**：根据解析出的全宗、年度、保管期限等维度，利用 `pathlib` 在操作系统底层自动递归创建 4 级/5 级目录树。
- **一夹多页支持**：完美兼容同一档号的多页图片并发上传。所有页面统一归档至同一物理目录，且仅触发一次数据库“已数字化”状态回写。
- **安全与性能**：内置文件扩展名白名单校验，并使用 `aiofiles` 实现异步非阻塞 I/O，保障大批量并发上传时的服务器性能。

### 💾 2. 数据库状态同步与以物建账
- **模拟数据库**：采用轻量级的本地 `database.json` 文件进行数据持久化，确保服务重启后状态不丢失。
- **以物建账**：支持档案行业“先扫描后录入”的场景，当上传了数据库中不存在的档号时，系统会自动为其创建一条初始档案记录。
- **实时状态校验**：前端请求列表数据时，系统会实时（双向同步）探测硬盘物理文件是否存在。若用户手动删除了物理文件夹，系统会自动将对应的数据库状态重置为“未挂接”，杜绝“有账无物”的死链。
- **事务回滚**：上传与落盘包裹在逻辑事务中。如遇文件名包含 `ERROR` 的模拟崩溃情况，系统会自动清理刚刚落盘的“孤儿文件”，保证强一致性。

### 🔍 3. Elasticsearch 全文检索
- **云端集成**：已对接并测试通过最新的 Elastic Cloud (Serverless) 架构。
- **规则注入（同义词）**：在 ES 索引创建阶段配置了 `my_synonym_filter`，实现查询扩展。例如：搜索“学生”，系统会自动召回包含“学校”或“校园”的档案数据。
- **多维度过滤**：支持先根据“年度”、“保管期限”等精确字段进行 `filter` 过滤，再进行文本 `match`，兼顾查询性能与准确度。
- **高亮与秒级开箱**：搜索结果返回带有 HTML 标签的关键字高亮（`<em>`）。前端展示绝对路径，并支持模拟直接调起本地 Windows 资源管理器的“秒级调阅”体验。
- **数据双写**：文件上传落盘时，元数据与生成的 OCR 文本不仅写入 MySQL(JSON)，还会同步双写 (Sync) 到 ES 索引中。

## 技术栈选型

- **后端框架**: FastAPI (Python 3.11+)
- **异步 I/O**: `aiofiles`
- **前端页面**: HTML5 + 原生 JavaScript + Jinja2 模板引擎
- **持久化**: 本地 JSON 文件 (`database.json`)
- **搜索引擎**: Elasticsearch 9.4.0 (Serverless 版本客户端 `elasticsearch>=8.12.0`)

## 目录结构说明

项目遵循领域驱动设计的模块化拆分：

```text
文件分件/
├── auto_filing/           # [核心模块] 自动分件与落盘
│   ├── archive_logic.py   # 解析规则、相对路径计算、异步物理落盘逻辑
│   └── database.py        # 模拟 MySQL 数据库、以物建账、路径回写、硬盘实时探测
│
├── search_engine/         # [核心模块] 全文检索与 ES
│   └── es_service.py      # Elasticsearch 云端连接、索引构建、查询规则、检索方法
│
├── archive_data/          # 真实的落盘绝对根目录（系统动态生成，可随时删除测试自愈）
├── database.json          # 模拟数据库持久化文件
├── requirements.txt       # Python 依赖清单
├── main.py                # FastAPI 主入口与路由控制器
└── templates/
    └── index.html         # 包含上传交互与检索界面的前端控制台
```

## 如何运行项目

### 1. 环境准备
确保您的电脑上已安装 Python 3.9 或以上版本。

### 2. 安装依赖
打开终端，进入项目根目录，运行以下命令安装所需依赖库：
```bash
pip install -r requirements.txt
```

### 3. 配置 Elasticsearch (可选)
如果需要测试全文检索功能，请打开 `search_engine/es_service.py`，将以下两项替换为您真实的 Elastic Cloud 凭证：
```python
ES_ENDPOINT = "https://your-project-url.es.asia-southeast1.gcp.elastic.cloud:443"
ES_API_KEY = "Your-Encoded-API-Key"
```
*(如果不配置，系统会自动跳过 ES 初始化，仅保留自动分件功能运行)*

### 4. 启动服务
在终端中执行：
```bash
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

### 5. 访问控制台
打开浏览器，访问：[http://127.0.0.1:8000/](http://127.0.0.1:8000/) 即可看到操作界面。

## 常用测试用例

- **正常分件测试**：准备两张图片，分别命名为 `WS-2024-D10-0311-001.jpg` 和 `KJ-JJ-2017-02-001-000.jpg`。在左侧批量上传，观察右侧表格状态变绿。
- **异常回滚测试**：上传一张名为 `FIN-2021-Y30-0112-ERROR.jpg` 的图片，观察日志提示“事务回滚，清理孤儿文件”。
- **状态同步测试**：去电脑文件夹里把生成的 `archive_data` 整个删掉，然后在网页点击“刷新数据库状态”，观察所有绿色状态瞬间重置为红色“未挂接”。
- **同义词检索测试**：在右侧搜索框输入“学生”，点击检索，查看是否成功带出了只包含“学校”或“科研”字眼的文章。
